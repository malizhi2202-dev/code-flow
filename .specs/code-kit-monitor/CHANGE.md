## Why（为什么做）

code-kit 在运行过程中产生大量中间状态（工作流节点切换、skill 调用、token 消耗、产物文件、专家团投票），当前所有信息散落在 `.specs/` 目录和终端输出中。开发者面临三个核心痛点：

1. **「跑到哪了？」** — 清窗后不知道当前 change/阶段/task，也不知道门禁过了没
2. **「烧了多少？」** — 不知道全链路 token 消耗，预算是否快用完
3. **「卡在哪？」** — 看不到专家团投票状态、哪个 task 被标了人工、哪个阶段停留过久

需要一个工业风本地监控面板，聚合展示 code-kit 全链路状态。

## What（做什么）

一个**本地 Web 监控面板**（前端 SPA + 轻量数据聚合服务端），从 `.specs/` 目录和 git log 解析数据，为**单人开发者**和**团队 lead** 提供 code-kit 工作流的可视化监控。

### v1 范围（本次必做）

| 维度 | 内容 |
|---|---|
| **工作流全景** | 活跃 change 列表、当前阶段、进度、中断任务醒目提示 |
| **产物查看** | 所有 `.specs/<id>/` 下的产物**只读查看 + 批注**（不做编辑，避免人机冲突） |
| **专家团审核** | 6 Gate 投票状态展示（含每角色 ✅/❌ + 投票理由）、平票/争议标记 |
| **Task 执行** | 波次进度、`<auto>` 自动化/人工分布、通过/失败/重试 |
| **Token 概览** | 顶部聚合计数器（总额/阶段额），不展示单次请求详情 |
| **搜索 & 过滤** | 按 change-id / 关键词 / 状态过滤 |
| **Git 安全网** | 最近 safety checkpoint 位置、回滚距离 |
| **数据一致性** | `.specs/` 产物自洽校验（TASK vs SUMMARY 数量、AC vs TEST 覆盖对齐） |
| **安全事件** | 秘钥扫描告警醒目标记 |

### v2（下次再说）

| 维度 | 内容 |
|---|---|
| 产物编辑 | 在线编辑 markdown + 冲突检测 + 变更追溯 |
| 质量趋势 | Critical/Major/Minor 历史曲线、T-FIX 累计 |
| 瓶颈分析 | 每阶段停留时长、门禁等待耗时、change 总周期 |
| 成本分析 | token 预算 vs 实际对比、模型调用分布、预算超支告警 |
| 工具链监控 | brooks-lint / uipro / impeccable 在线状态 |
| 实时推送 | WebSocket 实时状态推送（v1 用文件轮询） |
| 跨 change 对比 | 团队 lead 视角：谁的自动化率最高、哪个 change 卡最久 |

## 视觉调性（前端项目必填）

- **选定**：3️⃣ 工业（Industrial）
- **理由**：监控面板核心场景是数据密度——工作流拓扑、token 时序、任务矩阵、投票状态，需要等宽字体 + 冷色温 + 暴露网格支撑「一屏呈现多维度而不乱」。暗色主题默认，亮色不低于 WCAG AA
- **参考产品**：Grafana、Datadog、Bloomberg Terminal
- **明确排除**：玩具（太轻浮）、奢华（监控不需要）、有机（不适合数据密集型）、复古未来（不专业）

> 附带条件（来自 G1 门禁）：暗色/亮色双主题，暗色默认

## 范围排除（这次不做）

- **不做**产物在线编辑（v1 只读 + 批注，编辑留 v2）
- **不做**实时 WebSocket 推送（v1 用文件轮询）
- **不做**公网部署/多用户权限系统（仅本地 localhost）
- **不做**移动端适配（桌面优先）
- **不做**历史趋势图表（需要积累数据，v2 再做）
- **不做**跨模型 spot-check 结果聚合（先做基础监控）

## 验收线（粗粒度，不是 AC）

- 本地启动面板后，能看到当前仓库所有活跃 change 的阶段/进度/task 状态
- 能看到每个门禁的 4 角色投票结果和理由
- Token 消耗以聚合数字展示在顶部，不暴露单次请求内容
- 中断任务有醒目入口，点击能看到恢复指引

## 风险与未知

- **数据源稳定性**：`.specs/` 文件格式由 code-kit 的 prompt 控制，如果 prompt 改版导致产物格式变化，面板解析可能失效 — 缓解：解析逻辑与 code-kit 版本绑定
- **性能**：大型仓库 `.specs/` 下可能积累大量 change 产物，文件扫描可能变慢 — 缓解：v1 限制展示最近 90 天的 change
- **部署环境**：仅本地 localhost，不设访问控制；用户明确表示仅本地演示用途

## Why

code-kit 运行中产生大量中间状态（工作流节点切换、skill 调用、token 消耗、产物文件、专家团投票），信息散落在 `.specs/` 和终端输出中。开发者面临三个痛点：

1. **「跑到哪了？」** — 清窗后不知道当前 change/阶段/task，也不知道门禁过了没
2. **「烧了多少？」** — 不知道全链路 token 消耗，预算是否快用完
3. **「卡在哪？」** — 看不到专家团投票状态、哪个 task 被标了人工、哪个阶段停留过久

需要一个本地操作台，聚合展示 code-kit 全链路状态。

## What

一个**本地 Web 操作台**（前端 SPA + 轻量数据聚合服务端），从 `.specs/` 目录和 git log 解析数据，为开发者和团队 lead 提供 code-kit 工作流的可视化管理和监控。

### v1 范围

- 工作流全景：活跃 change 列表、阶段、进度、中断提示
- 产物管理：阶段时间线三栏布局，查看+编辑+自动备份
- 专家团审核：8 Gate 门禁矩阵表（Gate × 角色）
- Task 执行：波次进度、自动化标记、重试次数
- Token 概览、搜索过滤、Git 安全网、数据一致性校验
- 工作流编辑器：阶段增删改排序 + 门禁专家团分配
- 角色管理：12 角色查看/编辑性情/新增/删除
- 文档编辑器：直接编辑 code-kit prompts/templates/rules
- 项目切换器：多项目隔离
- Grafana 风格暗色操作台 + 左侧导航布局
